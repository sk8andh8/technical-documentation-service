# ТехАрхив — электронный архив технической документации

Десктоп-приложение для распознавания (OCR) основной надписи инженерных
чертежей по ГОСТ 2.104, ведения структуры изделия (дерево и граф
применяемости) и хранения PDF-документов в PostgreSQL.

Интерфейс на русском языке, построен на PyQt6.

---

## Возможности

- **Журнал документов** — таблица с ID, обозначением (ГОСТ 2.201),
  наименованием и прикреплённым PDF. Поиск по всем полям, переключение версий.
- **OCR по ГОСТ** — распознаёт обозначение изделия и первичную применяемость
  (графа «Перв. примен.») из фиксированных областей листа. Если Перв. примен. уже
  есть в архиве — связь применяемости создаётся автоматически.
- **Дерево изделия** — иерархия применяемости; колонка «Тип»
  (Изделие / Сборка / Деталь) по положению в структуре.
- **Граф изделия** — структура без дублирования узлов: сплошные синие рёбра —
  первичная применяемость, пунктирные фиолетовые — заимствование.
- **Версионирование** — у каждого элемента несколько версий (начиная с v1),
  связи общие для всех версий.
- **Встроенный просмотр PDF** + экспорт файла на диск.

---

## Требования

| Компонент | № изменения | Назначение |
|-----------|--------|------------|
| Python | 3.12+ | среда выполнения |
| Docker + Docker Compose | любая актуальная | PostgreSQL 15 |
| Tesseract OCR | 5.x | распознавание отсканированных чертежей |

> OCR через Tesseract нужен только для **отсканированных** PDF. Для PDF с
> электронным текстом (как `example_document.pdf`) распознавание работает без
> Tesseract — текст берётся напрямую из нужных областей листа.

## Установка и запуск (из исходников)

### 1. Установить зависимости Python

```bash
pip install -e .
```

Или вручную:

```bash
pip install pyqt6 sqlalchemy psycopg2-binary PyMuPDF Pillow pytesseract
```

### 2. Поднять базу данных (PostgreSQL в Docker)

```bash
docker-compose up -d
```

Контейнер слушает порт **5439**. При первом запуске `database/init.sql`
создаёт схему, ограничения (запрет циклов, один первичный Перв. примен.) и
наполняет демонстрационными данными.

Строка подключения (захардкожена в `database.py`):

```
postgresql://std_user:std_secure_password@localhost:5439/std_archive_db
```

### 3. Установить Tesseract OCR (для сканов)

- Windows: установить в `C:\Program Files\Tesseract-OCR\` (путь захардкожен в
  `service/ocr_service.py`). При установке отметить русский языковой пакет
  (`rus`).
- Скачать: <https://github.com/UB-Mannheim/tesseract/wiki>

### 4. Запустить приложение

```bash
python app.py
```

При старте таблицы создаются автоматически (`create_all`), а недостающая
колонка `designation` добавляется идемпотентной мини-миграцией
(`ensure_schema`) — на уже существующей базе данные не теряются.

## Сборка в исполняемый файл (.exe, Windows)

Для демонстрации можно собрать один `.exe` через PyInstaller.

```bash
pip install -e ".[build]"     # ставит pyinstaller
python assets/make_icon.py    # (необязательно) пересоздать иконку
pyinstaller techarchive.spec
```

Готовый файл: **`dist/ТехАрхив.exe`**.

### Что НЕ входит в .exe (ручная настройка у пользователя)

`.exe` — это только клиент. До запуска нужно:

1. Запустить PostgreSQL: `docker-compose up -d` (порт 5439 должен быть свободен).
2. Установить Tesseract OCR (только если требуется OCR сканов).

Это приемлемо: сборка предназначена для демонстрации, а не для дистрибуции.

## Структура проекта

```
app.py                       Точка входа (main): миграция схемы, иконка, запуск окна
database.py                  Engine/Session/Base + ensure_schema (мини-миграция)
resources.py                 Имя приложения и пути к ресурсам (совместимо с PyInstaller)
database/init.sql            Схема + ограничения + демо-данные (первый запуск Docker)
models/                      ORM-модели Item (id, version, designation, name, file_data) и ParentChild
repository/                  CRUD + проверки (циклы, один первичный Перв. примен.)
service/
  ocr_service.py             Чтение фиксированных областей штампа (электронный текст + OCR-fallback)
  gost_parser.py             Разбор обозначения и «Перв. примен.» из областей листа
ui/
  main_window.py             Главное окно: вкладки Журнал / Дерево / Граф
  item_form_dialog.py        Форма элемента (обозначение и наименование — отдельные поля)
  parent_child_dialog.py     Создание связи применяемости
  pdf_viewer_dialog.py       Встроенный просмотр PDF (QPdfView)
  product_tree_widget.py     Вкладка «Дерево изделия» (+ колонка Тип)
  product_graph_widget.py    Вкладка «Граф» (без дублирования узлов)
  workers.py                 Фоновый OCR-поток
assets/
  make_icon.py               Генератор иконки
  app_icon.ico / .png        Иконка приложения
techarchive.spec             Конфигурация сборки PyInstaller
```

---

## Замечания

- Парсер ГОСТ опирается на фиксированные пропорции листа: нужные графы берутся
  из заданных областей (в долях страницы), что одинаково работает для
  электронных и отсканированных чертежей.
